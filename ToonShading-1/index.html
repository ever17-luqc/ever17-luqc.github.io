前言

须知写在最前：本文是基础篇，只涉及基础的日式卡通渲染，日式卡通因为和PBR完全不是一个概念，trick非常多，本文只涉及常见的做法。

图形学中有一个非真实感渲染（NPR），卡通渲染作为其中的一个分支，由于国内二刺猿手游的盛行相较其他内容简直如鱼得水，不过也由于本身并不基于物理，没那么多“奇形怪状”的公式，相较PBR渲染更加考验美术把控。由于二刺猿文化的兴起，为了拿走死宅钱包里的存货，国内二刺猿手游风行了一段时间了，各路厂商或多或少都整了点二刺猿方向的尝试。国内比较有代表性的米哈游的崩3或者原神

库洛的战双

​

​ 日产的作品，最有代表性的就是罪恶装备系列了（算是近年来的卡通渲染的祖师爷级别的了，比如崩3在基础的着色上就沿用了罪恶装备的ilm贴图做法，后续相关的会讲到）

同样比较知名的日厂游戏还有Cy的碧蓝幻想

之前还看到有人分享万代南梦宫的蓝色协议的渲染：

 **总的来说各个厂商都有自己的风格特色，但是从原理上大同小异，只不过都有自己的做法上的改进或者说针对项目特点做特定的修改**。

​ 这篇文章将会简单说说关于日式卡通渲染的基础以及背后的原理。

入门级卡渲从这里开始

明暗着色基础

关于卡渲以及相关的内容我自己的理解

在讲卡渲的明暗着色之前，必须先讲讲最基础的东西：Animation Style或者说是Toon和Physically Based的本质区别。动漫也好插画立绘也好，这都算是基于创作者理解的世界的提炼总结，对特征的精简以及修改，最典型的就属美式卡通那类，对于器官的极度提炼夸张。如下图：

这么讲其实也还是太过于笼统，因为不仅仅是形状上的精简和夸张修改，在细节密度上也有别于现实或者说Physically based。我可以举一个更加代表性的例子：

拿守望先锋来说，暴雪在制作时候是基于Physically Based Rendering（PBR）的，这点就能很好论证了我前面的观点，卡通渲染≠抛弃PBR的便捷性和准确性，他也可以是细节密度的减小。比如守望里的金属如上图的大锤给人非常规整的感觉，但是现实里的金属如果是有点年岁了的会呈现如下的结果：

守望先锋在PBR的基础上减少了粗糙度、法线、金属度的信息密度，形成了这种基于物理渲染的卡通提炼。

​ 也就是说卡通渲染和PBR虽然有着明确的边界，但是完全可以做到合理结合（虽然这篇文章将的更多偏向日式卡渲），战双就在高光中引入了PBR的高光项，让部分材质的金属感更加符合现实做到了不错的融合。本文不会涉及PBR的深入部分，因为本文重点是卡渲后续就带一下，后续有时间会开一个PBR的文章。

​ 但是如果纯PBR或者PBR为主的流程 （例如守望先锋），会让制作流程大大缩短，因为PBR的流程相关DCC一站到位，基本没过多需要修改的地方，对于3D美术也会非常的友好，而传统的卡渲只要有自己的定制化，必然需要一段时间的美术团队磨合（上年底毕业的时候公司在做的二刺猿项目当时就因为这点使用了衣服PBR皮肤卡渲的做法来大大削减开发成本和时间，当然也有风格上的因素需要写实一点的衣服，算是一举两得吧）。

讲完卡通渲染的理解，讲讲日式的卡渲的特点。如果是学过原画或者了解过相关内容的朋友应该了解，现在原画分成了两个大的种类：平涂和厚涂。平涂在日系二次元里可以理解为赛璐璐，也就是常见的描边+明显的明暗交界+相对于厚涂更加明快的颜色。区别于厚涂，日式卡通颜色选用会偏向于亮、高饱和，过度或者说是明暗交界线会更加硬朗极致点就是后面会说的cellshading，而明暗交界的软硬算是界定半厚涂风格的赛璐璐和纯赛璐璐的区别。

cell shading：纯赛璐璐的开始

关于Cell Shading ，为了得到卡渲里的强烈的明暗划分，我们需要去量化他的灰度，对他的灰阶进行处理，比如灰<0.5为黑色而相反是白色。

做法很简单，基于NDotL（也就是常规的兰伯特光照用法线方向去点乘光线的方向）的 做一次Step得到强烈的明暗边界划分（如果是有羽化的边缘或者需要做假的皮肤透光后续会继续讲到）。

这里*0.5+0.5就是沿用了半兰伯特的做法，让暗部不至于纯黑，让明暗着色更加向亮部迁移（实际使用并不一定需要是0.5，可以按照项目风格修改甚至是暴露参数给美术）。

rampMap引入

valve在《军团要塞2》首先开始使用这个做法，我们在拿到光照强度结果后（无论是step的还是smoothstep的），本身只是灰度信息或者说就是强度信息，军团要塞的制作团队把这张只有X轴记录颜色信息的贴图引入了进来，称其为rampmap（如果有使用过SD的朋友可以完全理解为gradientmap）

，估计各位看到这张图就猜到了些什么，我们算的NdotL本身就是对应的光照方向法线方向的光照强度，那么我们以NdotL（其实是NdotL*0.5+0.5，因为NdotL在【-1,1】区间，我们UV的x轴需要【0-1】）作为X轴坐标，去采样这张rampmap不就完成为了美术指定不同明暗的颜色变换吗。具体逻辑其实就是一个颜色映射技巧。如下图

起夜级李姐-ilm贴图

更加官方的内容可以参考2015年GDC上Arc System Works’ RED team的分享“GuiltyGearXrd’s Art Style : The X Factor Between 2D and 3D”GuiltyGearXrd’s Art Style : The X Factor Between 2D and 3D - YouTube，干活太多，网上也有太多的分析解读，这里只简单谈谈ilm贴图部分。

他们的基础shading部分依然是沿用了celshading，这点现在大多纯日式渲染都是这个路子。因为如果smoothstep没那味儿，不够二，因为传统的日本动画也好漫画也好一般不会去做阴影过度（节省成本和时间）。

然后就是关键点，他们的ilm贴图，其中RB通道控制高光的强度和范围比较常规，G通道比较酷炫，整了一个固定阴影通道，配合顶点色形成一个阈值区域，如果漫反射强度小于这个阈值将变成固定阴影区域，这种做法好处在于，如果漫反射贴图上画阴影显然是错误的做法，相当于漫反射贴图=漫反射*暗部阴影，这样在计算玩光照强度后乘上漫反射颜色等于做了两次暗部计算，会导致结果完全出乎把控。

这个G通道也可以理解为PBR的AO的漫反射版本，AO遮蔽的是环境光，而ilm的G通道为的是动漫里的固定阴影区。

高光部分

​ 高光的做法暂时还比较常规，NPR里的修改高光的形状之类的做法以前试验过，但是效果非常不可控，这在于流程化的制作中是非常不利的。

    现在常见的做法还是ilm贴图RG通道控制高光强弱范围也就是类似simplelit的那套做法，还有就是引入PBR的高光项，比如现在常用的GGX高光项

​ 拿常规的blinn-phong高光结合ilm举一个例子

边缘光部分

现在比较常见的有基于深度图偏移的，也有常规的基于1-NdotV然后拿反向lambert遮罩的。

下图为了方便观察把边缘光拉到最亮：

这种基于1-NdotV的边缘光最大的问题就是硬表面会由于法线过于一致导致亮面很大很难看，可以采用深度图偏移的方法。

深度图偏移边缘光的做法会和基于深度的额发投影类似，后续会讲到基于深度的额发投影。

环境光部分

最常用的是IBL的那套做法，环境光漫反射部分可以用球谐光照来做，镜面反射部分基于粗糙度采用cubemap。具体做法细节会有许多差别，比如对应的系数由于我们没有粗糙度和金属度，我们可以用ilm的r.b通道去做代替。或者最简单粗暴的做法是除非金属度高去掉环境高光项，只保留球谐光照（常规的二次元内容比如动漫一般环境光也就是叠一个色，很少去做环境光的细节，强反射的除外，但是强反射市面上的大部分二次元游戏也大部分选择去手绘贴图或者干脆直接用matcap做）

描边部分

​ 描边部分有许多分支。

​ 最简单的就是画在贴图上的描边，这里GGX也用到了这个，但是GGX为了描边贴图去除锯齿UV都是展直横竖摆，对UV要求很高而且会因为这种UV导致贴图上不能有纹理否则纹理必然由于UV问题存在大量锯齿（因为为了凑这种UV导致UV拉伸非常严重）。

然后就是常见的沿着法线方向挤出剔除前方。

​ 屏幕空间描边存在本身的问题，因为屏幕空间描边无法让同场景物体描边的粗细起伏不同，都是统一的粗细，而且由于本身是基于算子算梯度的，导致了低频部分描边困难，高频部分有时候又会存在描边过于散的问题，最大的问题也就是因为这种不可控导致了用的比较少。《无主之地》系列采用的就是这种屏幕空间描边。

​ 基于geometry shader的描边实现起来有比较大的难度，具体思路就是拿到轮廓边、边界边、折缝边，然后在geometryshader里把边转换成描边面。这种做法具有一定工程难度，本人暂时没自己实现过，有兴趣的可以参考灵刃大佬的这篇Unity3D 卡通渲染 基于退化四边形的实时描边 - L-灵刃 - 博客园 (cnblogs.com)

​ 最烂的办法是类似边缘光的做法，只是把边缘光部分拉强，非常不推荐，效果简直无语。

​ 还有一些游戏做了一些自己的处理，比如二之国的基于顶点色在屏幕空间描边，其实原理和屏幕空间描边一致，只不过屏幕空间描的是屏幕的RT，这里他描的是自己的顶点色RT。

​ 以常见的挤出外描线为例（有些人为了让不同距离看到的描边粗细一致，会*w做反齐次变换，但是实测效果不理想就没写）：

这里让裁剪空间的Z值沿着摄像机的Z偏移来消除某些转折处描边的断开。

另外像类似于头发之类的由于必然会做硬边，导致顶点可能存在多个方向的法线，这样导致的挤出会裂开，解决方法可以是写插件把和合并的法线放到顶点切线数据里，但是如果游戏里用到了法线贴图那么由于切线空间放的不是切线了会出现问题。以及也有把计算的法线存到UV3的方法（静态物体烘lightmap需要用到UV2）。参考硬边外描边断边问题 - 知乎 (zhihu.com)

插播：基于深度图的投影

原理非常简单，我们单独渲染一张头发的深度RT，只需要占用一个R通道，脸部材质中开启这个宏，偏移采样这张RT和当前深度对比，如果当前深度大于RT深度则判定为头发投影区域，你看，原理还不是shadowmap那套吗。
